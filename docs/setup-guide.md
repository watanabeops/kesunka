# セットアップガイド

KESUNKAは https://watanabeops.github.io/kesunka/ で公開済みです。このドキュメントでは、公開済みのアプリをiPadなどの端末で使い始める手順と、更新時の注意点を説明します。

## 1. iPadのホーム画面に追加する

1. iPadの**Safari**（他のブラウザではなく必ずSafari）で https://watanabeops.github.io/kesunka/ を開く
2. アプリが正しく表示されることを確認する
3. 画面上部（または下部）の共有アイコン（四角から上に矢印が伸びたマーク）をタップ
4. メニューから「ホーム画面に追加」を選ぶ
5. 名前（KESUNKA）を確認して「追加」をタップ
6. ホーム画面にアイコンが追加されるので、そこから起動する

ホーム画面から起動すると、Safariのアドレスバーなどが表示されない、アプリのような画面で使えます。一度開いておけば、オフライン（機内モードなど）でも記録の追加・閲覧ができます。

---

## 2. PC・スマートフォンのブラウザでそのまま使う

ホーム画面への追加は必須ではありません。上記URLをPCやスマートフォンのブラウザ（Chrome、Safariなど）で開けば、通常のWebページとしてそのまま全機能を使えます。データはそのブラウザ内（localStorage）に保存されるため、次回も同じ端末・同じブラウザで開けば記録が残っています。

ブラウザのプライベート/シークレットモードで開いたり、閲覧データを消去したりすると記録が消える点、また別のブラウザ（例: SafariとChrome）で開くとそれぞれ別のデータになる点にご注意ください。

保護者用ビューア（KESUNKA MAMA）も同様に、インストール不要でブラウザから開けます。

```
https://watanabeops.github.io/kesunka/mama.html
```

## 3. 保護者との共有機能を設定する

保護者が別の端末から、お子さんの学習状況（今日/週の合計時間・科目別の内訳・目標達成率・連続記録）を確認できるようにする機能です。Firebase（Googleの無料サービス）を使います。

設定手順は [docs/firebase-setup-guide.md](firebase-setup-guide.md) を参照してください。この機能を使わない場合は設定不要で、アプリ自体は端末内保存のみで通常どおり使えます。

## 4. 動作確認チェックリスト

- [ ] 公開URLでアプリが正しく開ける
- [ ] iPadのホーム画面にKESUNKAのアイコンが追加されている
- [ ] ホーム画面のアイコンから起動すると、全画面（アドレスバーなし）で表示される
- [ ] 科目・時間を選んでセッションを記録すると、「今日の学習時間」に反映される
- [ ] 機内モードにしても、ホーム画面から起動して記録・閲覧ができる（オフライン確認）
- [ ]（共有を設定した場合）保護者側の画面が、子ども側の記録後しばらくして自動更新される

## 5. こまったときは

**ページが真っ白になる／404エラーが出る**
一時的な反映遅延の可能性があります。少し時間を置いてから再読み込みしてください。

**アイコンが表示されない・ホーム画面に追加できない**
`icons/` フォルダの中に `icon-192.png` `icon-512.png` `icon-512-maskable.png` の3つが正しく入っているか確認してください。

**iPadでホーム画面に追加しても、ブラウザのまま表示される**
Safari以外（Chromeなど）で開いて追加していないか確認してください。iPadでは必ずSafariから操作してください。

**保護者側が「データ待ち」のまま変わらない**
子ども側・保護者側の両方で、Firebase設定（JSON）と共有コードが一字一句完全に一致しているか確認してください。詳しくは [docs/firebase-setup-guide.md](firebase-setup-guide.md) の「動作確認のしかた」を参照してください。

## 6. アプリを更新するとき（開発者向け）

このリポジトリの `main` ブランチにファイルをコミットすると、GitHub Pagesが自動的に再ビルド・再公開します（数分かかることがあります）。GitHubの「Add file → Upload files」で該当ファイルを上書きアップロードし、コミットするだけで反映されます。

PWA版は端末に一部のファイルを保存（キャッシュ）する仕組みがあるため、更新後もiPad側にすぐ反映されないことがあります。その場合は、アプリを完全に終了してから開き直してみてください。それでも変わらない場合は、ホーム画面のアイコンを一度削除し、上記「1. iPadのホーム画面に追加する」をやり直してください。

なお、`sw.js` 内の `CACHE_NAME` はコンテンツを更新するたびに値を変更する必要があります（変更しないと、インストール済みのPWAが古いキャッシュを参照し続けてしまいます）。
