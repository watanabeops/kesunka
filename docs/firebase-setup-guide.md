# 保護者への共有機能セットアップ手順（Firebase）

KESUNKAの「👪 共有」機能は、Firebase（Googleの無料クラウドサービス）のFirestoreというデータベースを使って、子ども側の学習状況の要約（今日/週の合計時間・科目別の内訳・目標達成率・連続記録）を保護者側（KESUNKA MAMA）に一方向で送信します。日々のメモや、いつ何をやったかという詳細は送信されません。

無料枠（Sparkプラン）は1日あたり読み取り5万回・書き込み2万回・保存容量1GiBで、家族利用であれば十分すぎるほど余裕があります。

## 1. Firebaseプロジェクトを作成する

1. https://console.firebase.google.com を開く（Googleアカウントでログイン）
2. 「プロジェクトを追加」→ プロジェクト名を入力（例: `kesunka-family`）
3. Googleアナリティクスは今回不要なので、オフのままで構いません
4. 「プロジェクトを作成」

## 2. Firestore Databaseを有効化する

1. 左メニューの「構築」→「Firestore Database」を開く
2. 「データベースの作成」
3. モードは「本番環境モード」のままでOK（次の手順でルールを上書きします）
4. リージョンはお好みで構いません（例: `asia-northeast1`（東京））
5. 「有効にする」

## 3. セキュリティルールを設定する

1. Firestore Databaseの画面で「ルール」タブを開く
2. 内容を次のルールに置き換えて「公開」

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /studylog_shared/{shareCode} {
      allow read, write: if true;
    }
  }
}
```

**重要**: このルールは `studylog_shared` コレクションに限定していますが、認証は行わないため、共有コードを知っている（推測できる）人は誰でも読み書きできてしまいます。KESUNKA側で生成される共有コードは十分長いランダム文字列（推測はほぼ不可能）ですが、パスワードと同じように扱い、むやみに公開しないでください。

## 4. Webアプリを登録し、設定情報を取得する

1. プロジェクトの概要画面（歯車アイコン→「プロジェクトの設定」）を開く
2. 「マイアプリ」→ `</>`（ウェブ）アイコンをクリック
3. アプリのニックネームを入力（例: `kesunka-web`）し、「アプリを登録」
   - Firebase Hostingの設定は不要なのでチェックを入れなくてOKです
4. 表示される `firebaseConfig` の中身（`{ apiKey: ..., projectId: ..., ... }` の部分）をコピー

この設定情報（apiKeyなど）はFirebase・Firestoreへの接続先を示すものであり、パスワードのような秘匿情報ではありません。アクセス制御は手順3のセキュリティルール（共有コード）側で行われます。

## 5. 子ども側（KESUNKA）で共有をオンにする

1. https://watanabeops.github.io/kesunka/ を開き、目標エリアの「👪 共有」をタップ
2. 「共有をオンにする」にチェック
3. 手順4でコピーした設定をそのまま「Firebase設定」の欄に貼り付け
4. 「新しいコードを生成」をタップ（共有コードが自動生成されます）
5. 「コピー」で共有コードをコピーし、保護者に伝える（メッセージ等で送る）
6. 「保存」をタップ

## 6. 保護者側（KESUNKA MAMA）で確認する

1. https://watanabeops.github.io/kesunka/mama.html を保護者の端末・ブラウザで開く
2. 「⚙️ 設定」→「Firebase設定（JSON）」に、手順4と同じ設定を貼り付け
3. 「共有コード」に、手順5で子どもから伝えられたコードを貼り付け
4. 「接続する」をタップ

これで、今日/週の学習時間・科目別の内訳・目標達成率・連続記録が表示され、子ども側で記録するたびに自動的に更新されます（ページの再読み込みは不要です）。

## 動作確認のしかた

1. 子ども側でセッションを1件追加する
2. 保護者側の画面を数秒待つと、数値・科目別の内訳が自動的に更新されることを確認する
3. 更新されない場合は、子ども側の「👪 共有」画面の送信ステータス（エラーが出ていないか）と、両方の共有コード・Firebase設定が完全に一致しているかを確認してください

## 補足

- 保護者側（KESUNKA MAMA）は閲覧専用で、書き込みは一切行いません
- 共有される内容は「今日/週の合計時間・科目別の内訳（科目名と分数）・目標達成率・連続記録」のみで、メモの内容や、日付ごとの詳細な記録は送信されません
- 共有をオフにしたい場合は、KESUNKA側の「👪 共有」→チェックを外して「保存」
- Firebaseの設定情報・共有コードは、それぞれのブラウザのlocalStorageにのみ保存され、このリポジトリのソースコードには含まれません
- Firebaseプロジェクトの管理（設定の削除・請求情報の確認など）は https://console.firebase.google.com から行えます
